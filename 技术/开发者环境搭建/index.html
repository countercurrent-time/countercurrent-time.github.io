文件系统

Windows 默认 NTFS，Linux 默认 exFAT，Mac OS 默认。

Mac OS HFS+ 有 Time Machine 机制。

推荐兼容性好的 UDF，不过 UDF 缺少错误修复工具，而且 Windows 和 Mac OS 下对 UDF 的格式化处理不完全一样。

UDF的通用性比微软的NTFS、exFAT、ReFS都要强，跨平台的交互能力与FAT32非常接近。

C++环境

由于 Windows 下没有像 apt-get 一样做到完全集大全的包管理工具，而且 Mingw 很不方便，下载速度慢，所以推荐 nuwen 下的 Mingw，下载 mingw-17.1-without-git.exe（此前要安装 Git，建议始终从最官方的软件来源获取软件，如这里的 Git），开箱即用。

也可以下载 mingw-w64，这个和 Mingw 不同。建议安装最新的 x86_64-posix-seh，而不是 Download Latest Version，后者是在线安装包，下载非常慢。

注意：任何版本的 Mingw 都至少需要 600M 的空间。

下载后运行程序，它会启用 7z 自解压。直接选择 C 盘就可以了，它会自动解压到目录下的 MinGW 文件夹下。

编辑器概览

付费 IDE 可以到 Github 教育包 中看一看有没有。Github 会为学生提供大量的免费服务。

Guide：不属于 IDE；NOI Linux 提供。

Devcpp：开箱即用，易于上手；不支持插件；长时间未更新；对大文件支持不好；窗口繁琐；不支持正则表达式；自动补全麻烦；使用已经落后的语言 Delphi 编写；BUG 较多；内存占用小；会自动缓存头文件，可能导致忘记添加头文件造成的 CE。

Code::blocks：开箱即用，易于上手；不断更新；支持插件；使用 C++ 编写。

Atom：性能、体验比 vscode 差，低配电脑有掉帧的可能性。不过性能正在提高。

Sublime：收费，不过性能等一些方面比 vscode 优秀。

Gedit、Kate：轻量的文本编辑器；NOI Linux 提供 Gedit。

Visual Studio：体积庞大；功能强大；部分版本要收费。

vscode：免费，轻量，强大，扩展性强，易于学习；支持 WSL；功能多样；使用率最高。

notepad++：作者敏感；恰如其名只是 notepad++；为了不必要的节约能源自制了一层毫无意义的 UI；图标难看（个人体验）。

IDEA 系列：收费（不过 Github 学生包里有）；性能要求高，体积庞大。

typora：用于编写 Markdown；界面简洁，功能单一；不开源。

vim、Emacs：强大，上限高，扩展性强，自定义程度非常高，学习曲线陡峭；NOI Linux 都提供；如果不使用 qwerty 键盘则可能会较难使用（视具体键盘而定，同时 vim 将不再高效。例如 hjkl 不再集中于右手下，而如果仍使用 hjkl 原位置下的字母，则很有可能会与其它快捷键冲突）。vim 体积小，灵活度高，并且有 neovim 代替；Emacs 有 evil mode、org mode 等，体积庞大，接近于操作系统。

Far manager：文件管理器；只能在 Windows 下使用；不如使用 vim；乱码现象较严重。

综上，平时建议使用 vscode 或者 vim。

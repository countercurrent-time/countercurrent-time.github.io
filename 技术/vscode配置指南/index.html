vscode配置指南

发表于 1970-01-01 更新于 2020-10-01 分类于 技术
配置强大的vscode。

前置

完成 mingw-w64 或者 nuwen 的安装和 path 配置。

用 vscode 打开自己的专门用于写程序的文件夹，在这个文件夹里新建 .vscode 文件夹，新建 tasks.json, launch.json, settings.json。

安装插件

可参考：Visual Studio Code 如何编写运行 C、C++ 程序？ - 腾讯技术工程的回答 - 知乎

我的插件列表：

  • Bracket Pair Colorizer 2：用彩色标记括号，并显示光标当前在哪一对括号里。
  • Indent-Rainbow：用彩色标记缩进
  • C/C++：官方C++插件。
  • Python：官方Python插件。
  • Chinese (Simplified)：中文扩展。
  • filesize：显示文件大小。不过只有在文件保存时才有效。
  • Markdown All in One：markdown 键盘快捷键、目录、自动预览等。
  • Remote - WSL：针对WSL的插件。
  • Todo Tree：标记TODO，显示TODO列表。
  • Astyle：注意是 Chieh Yu 的那个。不过没有 C/C++ 自带的 clang-format 好用。

快捷键

这个直接看官方文档吧。

完全避免 vscode 默认使用的 UTF-8 与系统 GBK 的冲突

UTF-8 可以支持全球各种语言的各种字符，但是 Windows 在中文环境下默认使用 GBK。

开始->设置->时间和语言->其他日期、时间和区域设置->区域.更改日期、时间或数字格式->管理.更改系统区域设置->勾选使用 Unicode UTF-8 提供全球语言支持->重启

这样会造成 Dev-C++ 中文乱码，所以这个方法也可以强迫自己不再使用过旧的 Dev-C++。如果实在需要 Dev-C++，可以把 Dev-C++ 下的中文语言配置文件用 vscode 打开，然后以UTF-8保存。

因为目前 Windows 的 UTF-8 支持还在 Beta 阶段，所以将 Windows 设置为使用 UTF-8 会造成 FAT32 的 U 盘中的文件名含有中文且比较长（20 个字符左右）的文件无法读取、复制，只能删除。

设置

ctrl - p 执行 >Preferences: Open Default Settings (JSON)，可以打开默认设置；

在打开的文件夹下的.vscode文件夹里修改settings.json，可以只在打开此文件夹的编辑器应用设置。

代码片段

单击设置 -> 用户代码片段，可以自定义需要在文件中频繁插入的代码片段。

Markdown 代码片段不能直接使用，需要在 settings.json 中加入：

# vscode # 配置
程序设计与静态查错

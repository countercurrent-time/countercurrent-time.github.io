莫队笔记

发表于 1970-01-01 更新于 2020-10-01 分类于 OI笔记

前置知识：尺取法

莫队算法在莫队提出之前，已经被 codeforces 的部分大佬使用，但是莫队是第一个对莫队算法进行详细总结的人。

对于一些区间询问问题，我们可以构造一种暴力思路：对于两个相邻的区间询问，我们计算出前一个询问的答案之后，以类似尺取法的方法，将前一个询问的左右端点分别移动到后一个询问的左右端点（每次只移动一个下标的位置），并且在移动左右端点的过程中维护需要维护的信息。

容易证明，如果移动下标并更新答案的时间复杂度为 $O(1)$（查询时间复杂度不必为 $O(1)$，如需要保证莫队 $O(n \sqrt n)$（$n, m$ 同阶）的时间复杂度需保证单次查询时间复杂度最多为 $O(\sqrt n)$）上面的操作时间复杂度期望为 $O(mn)$，但上面的这种方式可以被优化。

莫队就是是以分块的方式将询问离线排序以优化移动左右端点的时间复杂度，通过类似尺取法的方法算出每个询问的答案的算法。莫队由于思想与代码都很简单被广泛应用。但有时出题人为了考察选手的真实水平，会对题目加上强制在线的要求。当然，我们也可以使用在线莫队的方法求解。

虽然莫队总结莫队时只提到了普通莫队的操作，但经过后来 OIer/ACMer 集体智慧的改造，莫队有了多种形式：

  • 普通莫队
  • 带修莫队
  • 树上莫队
  • 真·树上莫队
  • 二次离线莫队
  • 在线莫队（诗乃莫队）
  • 莫队套 bitset

P1494 [国家集训队]小Z的袜子/【模板】莫队

本题要求计算区间内选两种不同位置的颜色，取到同一种颜色的概率。根据组合数相关知识，答案为 $\frac{\sum_{c \in [l, r]} C^2_{cnt_c}}{C^2_{r - l + 1}}$（根据题目，这里使用左闭右闭区间）。

我们需要维护每种颜色对应的 $cnt_c$，以及 $\sum_{c \in [l, r]} C^2_{cnt_c}$。此时使用一般的其他数据结构很难维护，但莫队则很容易维护。

我们先将询问全部输入，并按以下规则排序：设定块长度，如果询问左端点不在同一块内则按左端点升序排序，否则按右端点升序排序。

然后我们记录当前左右端点指针 $l, r$，并用数组 $cnt$ 记录当前区间 $[l, r]$ 内每种颜色的数量，记录变量 $sum = \sum_{c \in [l, r]} C^2_{cnt_c}$。为了回答询问 $[l_i, r_i]$，我们先处理出 $[l_{i - 1}, r_{i - 1}]$ 的答案，然后不断将左右端点 $l, r$ 分别从 $[l_{i - 1}, r_{i - 1}]$ 移动到 $[l_i, r_i]$。每次当我们将左端点或右端点移动一个数组下标时，我们修改 $cnt$ 中对应下标的颜色数量。我们可以通过 $cnt$ 计算出 $C^2_{cnt} = cnt \times (cnt - 1)$，所以我们也可以同时维护答案变量 $sum$。

时间复杂度证明：

设块长度为 $maxn$。排序之后，对于左端点在同一块内的询问，左端点每次最多移动 $maxn$ 的长度，最多 $n$ 次；右端点最多从 $1$ 移动到 $n$，最多 $\frac{n}{maxn}$ 次。为平衡时间复杂度，$maxn$ 设为 $\sqrt n$，总时间复杂度为 $n \sqrt n$。事实上，莫队的询问越多，单次询问的均摊时间复杂度越低。

我们还可以用奇偶性排序优化莫队常数：

原理：左端点在奇数块时，右端点不断往右移；左端点在偶数块时，右端点不断往左移，同时可以处理偶数块部分的查询，而不是往左移之后再往右移处理偶数块部分的查询，理论常数为原来的 $\frac{1}{2}$。

当然，如果把每个询问区间的左右端点位置看成平面坐标系上的点的横纵坐标，我们就可以通过求曼哈顿最小生成树，保证从一个询问转移到另一个询问的过程中，左右端点的移动距离之和尽可能小。求解曼哈顿最小生成树有 $O(n \log n)$ 的算法，莫涛大神也给出过。这样看起来更优，实则我们可以构造出平面上的 $n$ 个点，使这些点的横纵坐标均在 $[1, n]$ 范围内，并且这些点两两之间的距离均大于等于 $\sqrt n$。所以实际上通过曼哈顿最小生成树和左右端点所在块排序两种方法转移询问的时间复杂度一样，而我们一般采用更好写、程序常数更小的后者。

SP744 LPERMUT - Longest Permutation

（其实这题的做法应该尺取法的，不过莫队本身就是尺取法的延伸，就不单独写题解了）

看到题目是难以直接维护的判定性问题，想到如何使用尺取法统计。同时由于 1 这个元素是所有答案的根本，所以我们使用尺取法统计时要以 1 为中心寻找符合题意的区间。

我们判定一段区间是排列的依据是：

  1. 区间内至少有 1 个 1。（不过这是我们要寻找的区间中心点，所以不用管）
  2. 设区间最大值为 $x$，则区间数的总和是 $\frac{x \times (x + 1)}{2}$。
  3. 区间最大值等于区间长度。

我们每次枚举 1 的位置，从这个 1 向左右分别扩展，则最大值可能在 1 的左边或者右边。我们先讨论最大值在 1 的右边的情况：只需要每次枚举右端点，把当前右端点和右端点左边的第一个 1 之间的最大值作为当前右端点对应的区间最大值，就可以找出左端点的位置了。然后根据上面的第二点性质就可以判断当前区间是否是全排列。如果不是，则当前右端点一定不能与其他左端点组成全排列区间。而对于最大值在左边的情况，我们只需要把整个序列翻转一下，然后重复上面的方法就可以了。

我们可能会想到用 RMQ 问题判断区间是否有重复元素（查询每个元素的前驱位置构成的序列的区间最小值），再用前缀和求区间的和，但是并不需要这么复杂。

我们维护一个左端点，用尺取法将左端点移到正确的位置就可以了。这里需要加一条限制：左端点必须在当前右端点左边的第一个 1 和第二个 1 之间。

看起来时间复杂度巨大，但实际上这种方法非常高效：当前右端点从第 $i$ 个 1 的位置移向第 $i + 1$ 个 1 的位置时，由于最大值单调不下降，所以左端点的右移次数最大为 $p_{i + 1} - p_i$，左移次数最大也是 $p_{i + 1} - p_i$。所有 $p_{i + 1} - p_i$ 的和最大为 $n$，所以这个算法的时间复杂度是 $O(n)$ 的。

P1903 [国家集训队]数颜色 / 维护队列 /【模板】带修莫队

带修莫队在普通莫队的基础上加了一维时间维，需要根据时间维变化对修改操作进行执行与撤销。具体来说，我们用时间维 $time$ 记录当前经过了第几次修改操作。时间维移动时，我们要根据时间维经过的修改操作，修改记录答案的数组，同时我们可以将当前左右端点和时间 $[l, r, time]$ 变换成以下形式：

$[l + 1, r, time]$
$[l - 1, r, time]$
$[l, r + 1, time]$
$[l, r - 1, time]$
$[l, r, time + 1]$
$[l, r, time - 1]$

时间的移动与左右端点的移动不同。由于时间维移动时会经过修改操作，所以我们在移动时间维时，要把经过的操作变成逆操作。因为下次时间指针经过同一时间时，时间维移动方向与当前时间维移动方向相反，进行的操作也应该是本次经过操作的逆操作。

带修莫队需要将块大小设为 $n^{\frac{2}{3}}$，以保证 $O(n^\frac{5}{3})$ 的时间复杂度。如果对带修莫队使用普通莫队的排序函数，则带修莫队的时间复杂度为 $O(n ^ 2)$。

P5906 【模板】回滚莫队&不删除莫队

对于删除与添加一者比较容易，另一者比较困难的题目，我们可以考虑回滚莫队。

回滚莫队类似莫队并维持 $O(n \sqrt n)$ 的时间复杂度，但只保留插入操作，但回滚莫队也是细节最多的一种莫队。回滚莫队在将询问排序后（这里不能使用奇偶性排序优化），将询问按照区间左端点所在块分块，并记录当前左端点所在块。对于同一块内的询问，如果左右端点都在同一块内则暴力枚举，否则回滚莫队将右指针不断往右跳（同一块内右端点递增），并且对于每个询问，将左指针从当前块右端点跳到询问左端点，最后合并左指针和右指针移动过程中记录的答案。

SP10707 COT2 - Count on a tree II

数区间颜色是莫队的经典操作，而这道题目把区间变成了树上的链，所以我们考虑怎么把树上的链转化成区间。这就是树上莫队的中心思想。

对于树上莫队，我们可以考虑把树拍平成欧拉序，这样就可以把树上问题转化为序列问题，而且对于区间中重复出现的节点，我们可以用一个 $vis$ 数组判断节点是否遍历过，以确定是调用 $add()$ 函数还是 $del()$ 函数，即对于欧拉序上的一段区间，如果其中重复出现同一个节点，则将其忽略不计。这样我们可以保证，对于节点对 $x, y$，我们可以确定我们取 $x, y$ 在欧拉序上的对应位置的一段区间区间，其中可以包含 $x$ 到 $y$ 路径上的所有节点，且每个节点只出现一次（在某些情况下，$lca(x, y)$ 不会被包含）。

P3674 小清新人渣的本愿

莫队配合 bitset 可以兼并两者的优点：莫队可以维护常规数据结构难以维护的区间信息，而 bitset 可以维护常规数据结构难以维护的判定、统计问题。而本题就是这样的一道题。

我们用一个 bitset $s$ 维护当前区间内出现过的数。对于减法，因为
$$b - a = x \Rightarrow a = b - x$$
所以我们直接判断是否存在 $s_i = true \And s_{i + x} = true$，即判断 $s \And (s >> x)$ 是否存在至少一个 1。

对于加法，我们考虑将其转化为减法的形式。因为
$$a + b = x \Rightarrow a = x - b = N - b + x - N = (N - b) - (N - x)$$
所以我们只需要维护另一个 bitset $s2$ 存储 $N - b$，再判断 $s \And (s2 >> (N - x))$ 是否存在至少一个 1。

对于乘法，两个相乘为 $x$ 的数中必定有一个小于 $\sqrt x$，我们直接暴力枚举就可以了。

P5355 [Ynoi2017]由乃的玉米田

上一题的加强版，需要考虑除法。我们可能会想到像乘法一样枚举除数求出被除数。当 $x$ 较大时枚举量较小，而 $x$ 较小时这个算法的时间复杂度是 $O(mn)$ 的，难以承受。

用 $S$ 表示值域分块的分界点，则 $x \ge S$ 时的时间复杂度为 $O(\max(\frac{n ^ 2}{\omega}, n \sqrt n, nS))$。

我们考虑 $S$ 较小时怎么处理。一种直观的方法是对于每一种 $op = 4$，$x$ 相等的所有操作一起处理。这时我们有两种方法：

  1. 对于每一个数 $a_i$，我们考虑寻找 $a_i$ 左边的第一个 $j$，满足 $\frac{a_j}{a_i} = x$ 或者 $\frac{a_i}{a_j} = x$，并设 $last_i = j$。对于查询的区间 $[l, r]$，我们判断 $\max ^ r _ {i = l} last_i \ge l$ 是否成立。问题转化为 RMQ 问题，使用 ST 表解决的时间复杂度为 $O(Sn \log n + m)$。实际需要使用指令集优化才可以通过。
  2. 对于这类判定性问题，我们考虑另一种离线的方法：把所有询问按照左端点排序，并维护一左一右两个指针 $l, r$。对于每一个询问左端点，我们把 $l$ 移动到左端点，并把经过的数从 bitset 里删掉。然后不断将右指针右移，知道 $[l, r]$ 有符合题意的数。我们可以用一个变量维护当前区间商为 $x$ 的数对数量。这个过程的时间复杂度为 $O(m\log m + Sn)$，但实际常数很小。

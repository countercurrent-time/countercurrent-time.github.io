分块笔记

发表于 1970-01-01 更新于 2020-04-10 分类于 OI笔记
说分块是优雅的暴力的人，都不理解分块。 --lxl

分块很多时候被作为一种数据结构使用，但分块形式非常灵活，没有定式，而且分块更多地被当成一种思想。分块为了平衡时间复杂度，其块大小往往与根号有关，而且理论复杂度比较高，但实际常数却很小。虽然分块题目无法直接使用模板，代码细节比其他数据结构多，但分块代码比其他数据结构简单，而且应用最广泛。（不过细节多的特点也要求了必须多加练习才能很好地使用分块，这也与平衡树等数据结构不同）

lxl 因为其难度极高的数据结构系列题由乃OI，以及其中包含的大量毒瘤分块题而广为人知。

分块和莫队使用左开右闭区间会方便很多，即使用 $[0, n)$ 表示整个区间。

实际使用时可能会出现 $n % maxn \neq 0$ 的情况，不过不用考虑，因为最后一块代表的区间 $[n - n % maxn, n)$ 总是零散块。

虽然二维数组往往更方便，但我的参考代码中大部分的 block 都是一维数组（早期码风异常）。

区间分块

对每一段 $[k \cdot maxn, (k + 1) \cdot maxn] (0 \le k \le \frac{n}{maxn})$ 分别处理（如排序）以优化整块查询的复杂度，查询时对零散块和整块分别处理，还要单独考虑左右端点在同一区间内的情况。

P3372 【模板】线段树 1

把序列分成 $\sqrt n$ 块，预处理出每一段 $[k \cdot maxn, (k + 1) \cdot maxn] (0 \le k \le \frac{n}{maxn})$ 的区间和，并对所有块设立 $\sqrt n$ 个标记；

对于区间 $[l, r]$，计算出 $l$ 所在块的右端点 $L$ 与 $r$ 所在块的左端点 $R$；

对于修改操作，直接修改 $[l, L]$ 与 $[R, r]$ 之间的数；而对于 $[L + 1, R - 1]$ 的区间，对其中的所有块打上修改标记。

对于查询操作，直接累加 $[l, L]$ 与 $[R, r]$ 之间的数，以及它们所在区间的标记；而对于 $[L + 1, R - 1]$ 的区间，对其中的所有块直接累加区间和及其标记。

P1972 [SDOI2009]HH的项链

由于毒瘤的 chen_zhe 卡掉了莫队和分块，所以需要到 SP3267 DQUERY - D-query 提交分块或莫队代码。

本题为经典的区间颜色数问题，可以使用树状数组、线段树或莫队离线解决，也可以使用主席树或分块在线解决。

先思考离线算法：对于排好序的区间，遇到一个颜色后，及时将前一个与这个颜色相同的位置删除。

分块的思路也如此：

预处理出整个数列中每个位置前面与该位置颜色相同的位置 pre[]，同时将该数组复制到 preb[]，然后对于每一块，将块所在区间的 preb[] 分别排序。

查询时，对于零散的位置 i，如果 pre[i] < l 则累加到答案；对于整块，在区间中二分查找第一个位置 mid，满足 preb[mid] < l。

预处理时间复杂度为 $O(n \log n)$，查询时间复杂度为 $O(\sqrt n \cdot \log n)$。（注意：$\because \log_2 \sqrt n = \log_4 n, \therefore O(\log \sqrt n) = O(\log n)$）

UVA12345 Dynamic len(set(a[L:R]))

本题为上一题附加单点修改的操作，比之前复杂一些。我们需要动态维护数组 $pre$ 与 $preb$。

对于单次修改 $a[x] = y$，最多有三个位置的 $pre$ 可能发生变化：

  1. $pre$ 值为 $x$ 的数；
  2. 值为 $y$ 的 $x$ 右边第一个数；
  3. $x$ 本身。

为了动态插入、删除、查询对应一个权值的所有数的位置，我们需要对于每个值维护一个 $set$ 保存这个值在数组上的坐标。修改时修改上述三个位置的 $pre$ 值，并对三个位置所在块的全部 $preb$ 进行重构（从 $pre$ 复制再排序）。时间复杂度 $O(n \sqrt n \log n)$。

P4135 作诗

虽然莫队做起来很轻松，但是这题强制在线，如果不考虑在线莫队的话没法做。

首先考虑暴力：直接统计区间内的所有数的个数。但很容易想到一个常数优化的方法：由于每碰到一个数，该数的个数奇偶性会发生变化，所有可以用 bitset 预处理一些区间内数字个数的的奇偶性，查询时直接异或。为了平衡时间复杂度，采用分块的方法，处理 $[0, k \cdot \sqrt n)(k \in \mathbb{Z})$ 的 bitset 前缀和。

但这样只能计算出奇数个数的数。所以我们使用前面的分块在线区间求颜色方法求出区间颜色数，减去奇数个数的数即为答案。

预处理时间复杂度 $O(\frac{C \sqrt n}{\omega})$，单词查询时间复杂度 $O(\sqrt n \cdot \log n + \frac{n}{\omega})$。实际不开 O2 时会超时（加上快读可能好一些）。

CF911G Mass Change Queries

老师讲这道题时用的是线段树。一段时间后我从零开始做这道题时，想了 1h 多线段树的 push_down() 函数怎么写才能去掉 100 这个常数，然后才发现这题可以直接分块碾过去。自己很喜欢的数据结构竟然没有第一时间想到。

这题使用分块思路还是很简单的。

由于不能暴力修改，而普通的数组或者并查集又不能维护数与数之间的映射关系。所以我们为了使用延迟标记维护数的映射关系，需要考虑更加暴力的延迟标记维护方法：vector 启发式合并。

然后因为这个东西放到线段树上维护会常数巨大且非常难维护，所以我们考虑分块。

设最大值为 $size$，分块大小为 $maxn$，先对于每一块开 100 个 vector，然后使用两个函数：

  • merge() 即对于修改 l r x y 按秩合并同一块中的 x, y 对应的两个 vector，时间复杂度均摊 $O(\log size)$，且常数较小，一般跑不满。
  • assign() 即对于一整块，把 vector 维护的数的映射关系下传到单个数上，然后把 vector 恢复原状，时间复杂度 $O(maxn + size)$。初始时 vector 中 $v_{i, j} = {j}$。总的时间复杂度为每次修改 $\log size \frac{n}{maxn} + maxn + size$，最终对每一块 assign() 一遍的时间复杂度为 $O(n + \frac{n}{maxn} \times size)$。

由于分块常数小，虽然这题使用分块的时间复杂度很大，为 $O(n \sqrt n \log n)$，但实际上使用分块碾过去毫无压力，$maxn = \sqrt n$ 时用了 2.11min，最慢的一个点 1825ms，还算正常；而使用最优分块大小 $maxn = \sqrt {n \log size} + 1$ 时只用 1.58min，最慢的点 935ms。总用时大约是一般线段树用时的 3 倍。

如果实在要将这种方法卡到时间复杂度上界，可以考虑对这种方法进行更详细的时间复杂度势能分析，以找到构造数据的方法：

设一开始所有块都处于初态；

  • merge()：每一轮修改中，可以对一块消耗 $O(\log size)$ 的时间复杂度。如果对初态下的同一块使用了 size 次 merge()，则该块到达终态，不能再消耗时间复杂度（不过遍历到这一块的时间复杂度仍为 $O(1)$）。每一轮可以对任意块使用 merge()（实际为连续的若干块，不过这里可以不考虑）。
  • assign()：每一轮修改中，可以使一块恢复到初态。每一轮可以对最多 2 块使用 assign()。

所以如果要卡这种方法，可以设计这样的数据：先不断对区间 $1 n$ 执行可以将按秩合并卡到 $O(\log n)$ 的合并操作，然后再对区间 $[maxn * k + 1, maxn * k + 2 * maxn]$ 执行修改，这样就可以使用 $\frac{n}{2 \times maxn}$ 次操作使其恢复终态。

当然，以上的数据构造方法不能在实际情况下保证卡掉分块。因为分块 3s 跑 $n = 200000$ 的数据一般是可以过的，并且分块可以微调块大小，或者直接使块大小根据随机数波动，使别人根本没法卡。

这题里很多地方需要考虑 (i + 1) * maxn <= n 这样的边界情况，我还因此 Wa on 114 了几次。同时 CF 的 hack 机制非常毒瘤，导致 $n = 1, maxn = \sqrt {n \log n}$ 时会 RE，需要将 $maxn$ 改为 $maxn = \sqrt {n \log n} + 1$（不过设为 $maxn = \sqrt {n \log size} + 1$ 时不用考虑）。

整体分块

求出所有形如 $[m \cdot maxn,n \cdot maxn](1 \le m, n \le \frac{1}{maxn})$ 区间的答案，回答询问时合并 $[l , m \cdot maxn],[m \cdot maxn, (m + n) maxn]$ 与 $[(m + n) maxn, r]$的答案。这种方法一般只适用于没有修改的题目，对不具有结合律的性质效果极佳。（为了与题目一致，这里暂时使用了左闭右闭区间）

值域分块（根号分治）

如果数据的大小会影响程序效率，此时可以对 $[1,maxn]$ 与 $[maxn,n]$ 之间的数分别处理，即根号分治。事实上，算法竞赛中有三种常用的值域分治方式：整体二分、根号分治和基于 CPU 优化的 bitset 类分治。

一般来说，我们对于暴力处理慢的数据进行预处理，而对暴力处理快的数据直接暴力处理。

例题：

  • P3396 哈希冲突
  • UVA1383 Harmony Forever

P3396 哈希冲突

考虑到 $x$ 很小时，可以直接用一个 $max(x)^2$ 大小的表维护；而当 $x$ 很大时，可以直接暴力累加。

所以我们维护 $\sqrt n$ 大小的表，存储 $p \in [1, \sqrt n]$ 时的答案。在单点修改时对预处理的表和原数组进行修改，查询时，如果 $x < \sqrt n$，可以直接在表里查询；否则可以暴力累加。

UVA1383 Harmony Forever

$N = 5 \times 10 ^ 5$，考虑到 $p$ 较大时复杂度很低，所以考虑根号分治。

我们用一个数组维护模 $p$ 最小的数，其中 $p \le maxn$，同时将插入的数按顺序放进一个 set 里。当查找的 $p \le maxn$，则在数组中$O(1)$查找；查找的 $p > maxn$，则在 set 中对每个 $ik(k \in \mathbb{Z})$ 查找第一个大于等于 $ik$的数，复杂度 $O(\frac{N \log_2N}{maxn})$。

综上，$maxn = \frac{N \log_2N}{maxn}$ 即 $maxn = \sqrt{N \log_2N}$ 时复杂度最优。

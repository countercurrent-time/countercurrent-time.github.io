贪心笔记

发表于 1970-01-01 更新于 2020-10-01 分类于 OI笔记

贪心是一种特殊的算法思想，可以在题目含有以下性质时使用：

  • 子结构最优可以推出全局最优；
  • 状态无后效性，或者后效性可以被消除。

为了消除后效性，贪心经常配合堆、排序等使用。同时贪心没有定式，题目往往非常灵活，所以更需要不错的算法思维。

数学归纳法

例题：

  • POJ1700 Crossing River

POJ1700 Crossing River

用数学归纳法分类讨论的贪心。

题目大意：n个人过河，只有一条船，每次过两人回一人，给定每个人的过河时间，问最少用多长时间可以让所有人都过河。

样例是1 2 5 10，想了很久最快都要18秒，但是答案是17。

想明白了答案为17的原因：过1与2，回1；过5与10，回2；过1与2。

感觉是贪心，但想不出如何贪心，因为样例的贪心思路难看出。

然后查了题解。（毕竟咕了很久了）

过河有以下几种可能：

  1. 一人 $a$ 过河，所用时间为 $a$。

  2. 两人 $a,b$ 过河，$a < b$ ,所用时间为 $b$。

  3. 三人 $a,b,c$ 过河，$a < b < c$，$a,c$ 过桥，$a$ 返回，$a,b$ 过桥，时间总和为 $a + b + c$。

  4. 四人及以上过河，$a < b < \dots < c < d$。

    方案一：由最快的 $a$ 带$c,d$ 过桥：$a$ 带$c 过，$a$ 回来，$a$ 再带 $d$ 过，$a$ 回来。所用时间为$2a + c + d$，减少两人。

    方案二：由最快的a和次快的b过河： $a,b$ 过河，a返回；$c,d$ 过河；$b$ 返回。时间总和为$a + 2b + d$，减少两人。

    每次从两种方案中选一种，当人数减少到<=3，就成了前三种情况。

势能分析

例题：

  • SP348 EXPEDI - Expedition

SP348 EXPEDI - Expedition

大意：需要驾驶一辆卡车行驶 $L$ 单位距离，开始时卡车有 $P$ 单位汽油，途中有 $N$ 个加油站，第 $i$ 个加油站离终点距离为 $a_i$，可以加 $b_i$ 单位汽油。卡车油量无上限，问最少加几次油。

直接先往前开卡车，把途中的加油站油量放进堆里，每次没有油时把堆里油量最多的加油站取出来加油，复杂度 $O(n\log n)$。

可反悔的贪心

例题：

  • P1484 种树
  • P3045 [USACO12FEB]牛券Cow Coupons
  • P2748 [USACO16OPEN]Landscaping P

P1484 种树

先考虑较简单的情况：

当$k = 1$时，我们会选择最大的$a_i$；

$k = 2$ 时，我们先考虑最大的 $a_i$，此时有以下方案：

  1. 选择 $a_i$ 与 和 $a_i$ 不相邻数中的的最大数；
  2. 选择 $a_i$ 左右的数 $a_{i - 1}$ 与 $a_{i + 1}$；
  3. 选择 $a_i$ 旁边的数 与 和 $a_i$ 不相邻数中的的最大数；
  4. 选择 $a_i$ 不相邻数中的的最大数与次大数。

显然方案3和方案4没有方案1优，所以只有方案1和方案2可以考虑。

同时，我们发现我们推断时使用了贪心，所以我们考虑使用贪心思想。

然而，两种方案互相冲突，所以一般的贪心方法并不可行。

那么，我们考虑通过从 $k$ 到 $k - 1$ 的情况下如何解决问题。可以发现此时有两种选择：

  1. 从数列中去掉选中的数及这些数左右的数，剩下的数中选出最大值；
  2. 将一个已选中的数替换为左右的数，该数重新变回未选中状态。

我们的目标是设计尽量简明的状态，而第二种选择较为复杂，所以考虑将第二种选择转化为第一种选择可以表示的状态。

于是，我们考虑在选择当前最大值时，删除 $a_{i - 1}, a_i, a_{i + 1}$，再将 $a_{i - 1} + a_{i + 1} - a_i$ 放回 $i$ 位置，以便于解决 $k - 1$ 的子问题时可以反悔，如果重新选择此时的 $a_i$，就相当于选择了 $a_{i - 1}$ 与 $a_{i + 1}$，而且不能再次选择 $a_i$ 了。

为了方便维护每个元素的左右元素，我们建立一个链表维护元素的左右关系，并在选择当前最小值 $a_i$ 时，删除节点 $i, i - 1, i + 1$，建立权值为 $a_{i - 1} + a_{i + 1} - a_i$ 的新节点，将原来的 $a_i$ 累加到答案中，并且维护链表。

代码：

P3045 [USACO12FEB]牛券Cow Coupons

先尝试对使用优惠券后价格最低的 $K$ 头奶牛使用全部 $K$ 张优惠券，将钱减去她们的 $c_i$ 总和，然后用一个小根堆存储使用了优惠券的牛的 $p_i - c_i$，这样之后可以反悔。

然后考虑怎么反悔：当钱还没有花完时，每次考虑当前使用优惠券后价格最低的牛和不使用优惠券价格最低的牛，这样的话使用优惠券的牛 $i$ 的价格为 $c_i + q.top()$，不使用优惠券的牛 $j$ 价格为 $p_j$，两者取最小即可。

P2748 [USACO16OPEN]Landscaping P

先看到题目中的移动泥土，如果想要用移动泥土代替购买和泥土的操作，应该先每次尽量选出距离最小的泥土不够和泥土过多的花坛，将后者的泥土移动到前者。由于距离有绝对值，不好处理，想到分类讨论。

但距离绝对值最小的条件仍然不好处理。一种比较容易想到的思路是将所有泥土过多的花坛放到一个小根堆里，按照离它们最小的泥土不够的花坛的距离排序。但这种方法有不足，没有很好地处理状态后效性。

反例：

在这个例子中，$1, 5$ 号花盆泥土不够，$4, 7$ 号花盆泥土过多，按刚才的方法，我们会将 $4, 5$ 与 $7, 1$ 号花盆分别配对，但这种方法没有 $4, 1$ 与 $7, 5$ 的配对方法优，所以不可行。这种方法只能获得 16 分，不过这种方法在本题弱化版 P3049 [USACO12MAR]Landscaping S 中获得了 60 分。

然后考虑距离。因为状态有后效性，所以需要每次可以撤销之前的购买。

看到 $0 \le A_i, B_i \le 10$，应该想到把所有 $A_i, B_i$ 拆分成多件物品。

使用两个优先队列，分别存储泥土过多的 $i \cdot Z + Y$ 和泥土不足的 $i \cdot Z + X$，我们先假设对于每个花盆优先使用前两种方案，如果当前花盆泥土过多，则考虑从泥土不足的优先队列种选取最优的反悔方案；如果如果当前花盆泥土不足，则考虑从泥土过多的优先队列种选取最优的反悔方案。

时间复杂度 $O(10n \log (10n))$，实际因为 STL 优先队列常数大，所以跑到了将近 500ms。

解法看起来简单，但实际想出来并不容易，因为不少类似的贪心思路都是错的。

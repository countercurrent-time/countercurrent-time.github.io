树上问题笔记

发表于 2020-03-25 更新于 2020-10-01 分类于 OI笔记

总结由易到难的树上问题模板。

建图、建树

求 dfs 序、深度、子树节点个数等

dfs 序：树上 dfs 进入每个节点的序列，即树的先序遍历。dfs 序的长度为 $n$，每棵子树都对应 dfs 序上的一段区间，即 $[id_x, id_x + size_x)$。一般用于处理子树的信息，常见于 dsu on tree，树链剖分等。

欧拉序：又称括号序，dfs 进出一个节点时记录当前时间戳，并把时间戳加 1。每个节点会出现两次，因此欧拉序长度为 $2n$。一般用于处理有关一个节点到根节点之间路径的信息，同时可以处理一些简单的子树信息。常见于树上莫队、RMQ 求 LCA 等。

purfer 序列：其长度为 $n - 2$，可用于数列与无根树之间的互相转化，并且这种转化具有唯一性。具体可以参考这篇博客。

  • 无根树转数列：每次选取当前树上编号最小的叶子节点，删除此节点且将此节点所连接的节点加入数列末端，直到只剩两个节点。

  • 数列转无根树：根据 purfer 序列计算出每个节点的度数，节点 $x$ 的度数为 $x$ 在 purfer 序列中出现的次数加 1。然后第 $i$ 次选取 purfer 序列中的第 $i$ 个数与当前的最小标号度数为 1 的节点连边，并把两者的度数都减 1。执行 $n - 2$ 次，最后将两个度数为 1 的点连边。

求树的无权重心

树的无权重心性质：（以 $root$ 表示树的重心）

  1. 树上所有点到某个点的距离之和中，到树的重心的距离之和最短。
    $$\sum\limits ^ n _ {i = 1} dis_{i, root} = \min ^ n _ {i = 1} \sum\limits ^ n _ {j = 1} dis_{i, j}$$

  2. 一棵树可以有一个或者两个重心。如果有两个重心，则这两个重心在树上相邻。树上所有点到这些树的重心距离之和相等。

  3. 两棵树之间新加入一条边，新树的重心在两棵原树的重心连线上；同样的，一棵树的一条边断开，原树的重心在两棵新树的重心的连线上。

  4. 一棵树添加或删除一个节点，则树的重心最多移动一条边的位置。

树的直径

树的直径：树上最远的两点之间的距离，或者是树上所有路径中的最大值 $\max\limits ^ n _ {i = 1} \max\limits ^ n _ {j = i + 1} dis(i, j)$。

解法 1：深度优先搜索。从任一点 dfs 或者 bfs，找到离这个点最远的点 $p$，再从 $p$ 出发找到离 $p$ 最远的点 $q$，则 $dis(p, q)$ 即为直径。

最小生成树

定义无向图的最小生成树（Minimum Spanning Tree）为边权和最小的生成树。

性质总结：

  • 一个图可以有多个最小生成树，但是这些最小生成树的边权构成的可重集相同。
  • 一个图所有权值 $\le maxval$ 的边构成的图的所有最小生成树的任意两点的连通性相同。
  • 一些边集的并的最小生成树等价于这些边集的最小生成树的边集的并的最小生成树。
  • 最小生成树的唯一性：对于当前的最小生成树，如果无向图中有一条不是当前最小生成树的边，并且该边和最小生成树构成的无向图中唯一的环上有一条不是该边并且权值与该边相等的边，则该无向图的最小生成树不唯一。
  • 严格次小生成树：无向图中所有生成树的权值和中大小排名第二的权值和（不能与最小生成树权值和相同）。可以枚举无向图每一条不属于最小生成树的边 $(x, y, val)$，找到 $(x, y)$ 在最小生成树上的路径的一条边，满足边权严格大于当前边，用当前边替换掉该边即可。注意到该边的权值是严格大于，然而最小生成树上 $(x, y)$ 之间的路径上的最大值可能等于 $val$，所以我们需要同时计算严格次大边（出边权严格小于 $val$ 的最大边），在这种情况下用 $(x, y, val)$ 替换严格次大边。寻找最大边和严格次大边的过程可以用倍增、树链剖分或者 link cut tree 实现。
  • 最小瓶颈生成树：无向图中最大边权尽可能小的生成树。最小生成树一定是最小瓶颈生成树，最小瓶颈生成树不一定是最小生成树。同样，最小生成树上两点之间的路径即无向图上该两点之间的最小瓶颈路。但是两点之间的最小瓶颈路可能有多条，不一定每一条都在最小生成树上。
  • 最小树形图（最小有向生成树）：固定根的最小树形图可以使用朱-刘算法求解。

树上最近公共祖先（lca）及树上距离

模板：P3379 【模板】最近公共祖先（LCA）

从每个节点开始 dfs，可以暴力 $O(n ^ 2)$ 预处理树上两两距离，$O(1)$ 查询。另外也可以通过两点的父亲距离递推，不过本质差不多，而且 dfs 更容易写。

$O(n \log n)$ 预处理 lca，$O(\log n)$ 倍增查询：

ST 表完成 RMQ lca 可以做到 $O(n \log n)$ 预处理（可以通过正负 1 RMQ 实现 $O(n)$ 预处理，但常数很大），$O(1)$ 查询树上两点距离，空间复杂度 $O(n \log n)$。

树链剖分处理 lca 可以做到 $O(n)$ 预处理，$O(\log n)$ 查询。

Tarjan 可以 $O(n + m)$ 离线求 lca，但是常数很大，导致在 $n, m$ 处于十万级别的规模下时，Tarjan 与倍增 lca 实际运行时间差不多。不过 Tarjan lca 中的并查集具有特殊性质，使其中并查集的时间复杂度为 $O(n)$。具体可以见这篇论文，另外感谢这篇帖子里的各位大佬帮我解答了一些树上问题。

link cut tree 可以在动态维护树结构的同时支持查询最近公共祖先，原理可以看这篇博客或者这篇博客。

最短路建模

发表于 2020-02-17 更新于 2020-02-27 分类于 OI笔记

最短路常见的模型有差分约束、超级源汇、多源多汇、正反图、拆点、点分组等常见方法。为了保证时间复杂度，我们最常使用 dijkstra 算法；而对于数据范围较小的题目，我们更容易优先想到 Floyd 算法，并利用其动态规划。

然后，我们考虑将点分组。

我们先随机将关键点分为两组，分别与 $s$ 与 $t$ 连接边权为 $0$ 的边，然后跑 dijkstra，这样在多次运行后正确率趋于1。

但是这个方法与正解也差不多了。

正解：

点分组（官方题解）

假设我们把特殊点分成 $A, B$ 两个集合，新建 $s$ 连 $A$ 集合的所有点，边权 $0$ ，新建 $t$ 连接 $B$ 集合里的所有点，边权 $0$ ，那么 $s$ 到 $t$ 的最短路就是 $A, B$ 集合点之间的最短路的最小值。

那么对于 k 个特殊点，我们枚举二进制里的第 $i$ 位，把二进制第 $i$ 位是 $0$ 的点放在 $A$ ， $1$ 的点放在 $B$ ，用以上方法跑两遍最短路，分别以 $A$ 与 $B$ 连接 $s$ 与 $t$，以及分别以 $B$ 与 $A$ 连接 $s$ 与 $t$。

然后跑 $2 \log n$ 次最短路之后，所有最短路的最小值就是最终答案。

原理是，假设 $k$ 个特殊点里最近的是 $x$ 和 $y$ ，那么 $x$ 和 $y$ 一定有一个二进制位不一样，那么他们肯定在那次分组的时候被放进了不同的集合，从而肯定被算进了最后的答案之中最短路。

上面的复杂度为 $O(Tn\log^2 n)$。实际运行时需要使用快读与-O2优化才能通过。

正反图

考虑两个关键点对之间一定不会经过另一个关键点，所以在正反图分别跑 dijkstra，然后枚举所有边，找到最小的 $d_{from, 0} + val_i + d_{to, 1}$，时间复杂度为 $O(Tn\log n)$。

最短路动态规划

可以使用 SPFA，$d_{i, j}$ 表示到第 $i$ 个点，用了 $j$ 次免费机会的路径，队列里每个元素存储 $i$ 与 $j$。

$$dis_{to, j}= \begin{cases} \min({dis_{from, j - 1}, dis_{from, j} + val(from, to)}) & j > 0 \ \min({dis_{from, j} + val(from, to)}) & j = 0 \end{cases}$$

但本题卡 SPFA。虽然这题可以使用堆优化 SPFA，但堆优化 SPFA 也有可能被卡，所以最好使用分层图最短路。

分层图最短路

把原图的每个点 $a_x$ 拆为 $k$ 个点 $a_{x, 0 \dots k}$，从每个 $a_{x, i - 1}$ 到 $a_{to, i}$ 连接边权为 $0$ 的边，从 $a_{x, i}$ 到 $a_{to, i}$ 连接边权为原边权的边，初始时 $dis_{s, 0} = 0$，其他 $dis$ 为正无穷，然后跑 dijkstra，最终 $dis_{t, k}$ 即为答案。
